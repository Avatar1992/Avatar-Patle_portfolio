üìä Telecom NOC ETL Dashboard

Problem Statement

Telecom operations had a backlog of OSS alarm data arriving in heterogeneous formats. Teams lacked a centralized, real-time view to analyze alarm trends and SLA breaches.

Solution & Architecture

Built a streaming ETL pipeline that normalizes OSS alarms, persists structured data to PostgreSQL, and serves a real-time dashboard (Flask + Chart.js). Alerts are generated based on configurable thresholds and visualized for NOC operators.

PythonPostgresFlaskChart.jsDocker

Workflow

  • Ingest alarms (file / API) ‚Üí Normalize & enrich ‚Üí Store in PostgreSQL.
  • ETL jobs run continuously; failures retried & logged.
  • Flask service exposes dashboards; Chart.js renders real-time graphs.
  • Containerized services deployed with Docker for reproducibility.

Key Features

  • Real-time alarm normalization & visualization
  • Configurable alert thresholds for SLA monitoring
  • Retry & error handling for ETL stages

Challenges & Fixes

Handling variable incoming formats required building robust parsers and a lightweight schema registry. We implemented exponential backoff and dead-letter logging for failed records.

Outcome / Impact

Delivered a single pane of glass for NOC operations ‚Äî reduced alarm triage time and improved SLA visibility. The dashboard enabled faster incident response.

üöÄ CI/CD Pipeline (Jenkins & Docker)

Problem Statement

Solution & Architecture

Implemented Jenkins pipelines with declarative stages: checkout ‚Üí build ‚Üí unit test ‚Üí docker build ‚Üí push to registry ‚Üí deploy. Integrated webhook triggers and rollback strategies for failed deployments.

JenkinsDockerGitHubShell

Workflow

  • PR opened ‚Üí CI runs tests ‚Üí Merge ‚Üí Jenkins triggers release pipeline ‚Üí Deploy to staging ‚Üí Canary / Blue-Green deploy to prod.

Reduced manual release steps and decreased mean time to deploy. Increased deployment reliability and team confidence with automated checks and rollbacks.
